════════════════════════════════════════════════════════════════
WHAT NIMBUS IS
════════════════════════════════════════════════════════════════

NIMBUS · Emerald Slate v5 is a PyQt6 native desktop trading dashboard built for 
professional use on NSE (National Stock Exchange of India). It is NOT a web app, 
NOT a Streamlit prototype. It is a compiled Qt application running on macOS/Linux 
with a fixed 1480×900 minimum window, dark Emerald Slate theme, and JetBrains Mono 
as the primary monospace font.

The system is designed around a single professional trader's workflow:
- Pre-market: scan the universe for conviction setups (Scanner tab)
- Intraday: monitor individual symbols with deep analytics (Dashboard tab)  
- Macro context: understand which sectors have institutional money and momentum 
  (Market Context tab)
- Trade management: maintain a conviction-tiered watchlist (Watchlist tab)

Data sources in priority order:
1. KiteTicker WebSocket (Zerodha Kite) — real-time ticks, 4H bar aggregation
2. yfinance — 4H OHLCV polling fallback, daily sector data, options chain
3. NSE filings (BSE/NSE announcement APIs) — corporate events, bulk/block deals

CRITICAL RULES — DO NOT BREAK THESE
════════════════════════════════════════════════════════════════

1. NSE SESSION GUARD
   All real-time data fetches (price, options, tick streaming) must respect 
   NSE trading hours: 09:15–15:30 IST, Monday–Friday, excluding NSE holidays.
   Outside this window: use last cached data, mark status as STALE, do not 
   hammer yfinance with refresh requests. KiteTicker subscription should be 
   suspended outside session hours.
   This guard exists in data_manager.py and must be preserved in any refactor.

2. MFI MUST BE COMPUTED ON DAILY RESAMPLED FRAME
   MFI is computed via add_mfi() on the DAILY resampled frame, never on 4H 
   raw data. Reason: block deals and FII block buys create single-bar volume 
   spikes on 4H that make MFI meaningless. The daily resample averages them.
   Code path: compute_price_signals() → _resample_daily() → add_mfi() → 
   _compute_mfi_signals()

3. _resample_daily MUST ONLY INCLUDE PRESENT COLUMNS
   agg_dict must be built dynamically from columns that actually exist in df.
   The sector rotation pipeline builds OHLCV frames without Open column.
   Hard-coding "Open" in agg_dict causes KeyError → empty daily frame → 
   MFI = None for all sectors. Fixed version:
     _candidates = {"Open": ("Open","first"), "High": ("High","max"), 
                    "Low": ("Low","min"), "Close": ("Close","last"), 
                    "Volume": ("Volume","sum")}
     agg_dict = {k: v for k, v in _candidates.items() if k in tmp.columns}

4. ps SCOPE MUST BE INITIALIZED BEFORE TRY BLOCK IN SECTOR LOOP
   In _compute_sector_data(), ps is only assigned inside the conditional 
   entry viability block. If that block fails or is skipped, ps from the 
   PREVIOUS loop iteration bleeds into the current row's mfi_value.
   Always initialize: ps = None at the start of each ticker iteration.

5. s.get('key', default) DOES NOT GUARD AGAINST None VALUES
   If a key exists with value None, s.get('key', fallback) returns None, 
   not fallback. Use: val = s.get('key') followed by explicit None check 
   before any format string (e.g. f"{val:.0f}"). This caused the TypeError 
   crash in _sector_tooltip for mfi_value.

6. SYMBOL CHANGE = IMMEDIATE PIPELINE (Lesson 8.6)
   Never use QTimer.singleShot or any deferred trigger for pipeline execution 
   on symbol change. sidebar.symbol_changed must directly call 
   DataManager.refresh(symbol, fetch_options=True). Any refactor that adds 
   a delay or passive polling to this path is wrong.

7. pyqtgraph CONFIG BEFORE ANY ui/ IMPORTS
   pg.setConfigOptions(background=BG, foreground=MUTED, antialias=True) must 
   be called in main.py before any import from ui/ package. Qt widget 
   creation locks the config. Any refactor of main.py must preserve this order.

8. KiteTicker IS OPTIONAL — SILENT FALLBACK ONLY
   If kiteconnect is not installed or session is invalid, the app must work 
   perfectly on yfinance polling alone. No error banners, no broken state. 
   Log the fallback at INFO level, not WARNING. Only show a status update if 
   a session was previously valid and has expired.

9. SECTOR RS = PRICE-RATIO, NOT RETURN DIFFERENTIAL
   RS is computed as (sec_now/sec_N)/(bench_now/bench_N) - 1, not as 
   sector_return - benchmark_return. The price ratio is more stable for 
   ETF-vs-index comparison and avoids the compounding error of differential 
   returns over different base periods.
   pd.concat([sector, bench], axis=1, join="inner") is used to guarantee 
   date-index alignment across potential timezone differences.

10. BATCH DOWNLOAD WITH MultiIndex HANDLING
    sector_rotation.py uses yf.download(tickers, group_by="column") for 
    efficiency. The resulting DataFrame may be MultiIndex (field, ticker) 
    or flat depending on yfinance version. _extract_field() handles both 
    cases and must be preserved in any refactor.

11. MFI DIVERGENCE REQUIRES 3-STEP CONSECUTIVE DECLINE
    Bearish divergence (price near high + MFI falling) requires:
    - mfi[-1] < mfi[-3] < mfi[-6] (consistent trend, not noise)
    - Single-bar jump guard: abs(mfi[-1] - mfi[-2]) > 20 disables divergence
    - Only triggered when mfi_reliable=True (adv_cr >= 5 Cr)

12. SECTOR CACHE TTL = 900 SECONDS (15 MIN)
    fetch_sector_data(force=False) returns cached data within 15 minutes.
    Manual Refresh button passes force=True to bypass. showEvent uses 
    force=False to avoid hammering yfinance on every tab switch.

13. DATA QUALITY GATE FOR SECTORS
    MIN_VALID_BARS = 70, MAX_NAN_PCT = 0.05
    Any sector ETF with fewer than 70 valid trading bars or >5% NaN rows 
    is skipped entirely. This prevents thinly-traded ETFs from polluting 
    the rotation table.

ANALYTICS ENGINE
════════════════════════════════════════════════════════════════

PriceSignals dataclass (modules/indicators.py):
  # Daily bias
  daily_sma, daily_bias (BULLISH/BEARISH/NEUTRAL), daily_bias_pct
  
  # Bollinger Bands (20-period, 1.0 std)
  bb_position, bb_pct, position_state, upper, mid, lower, last_close
  position_state ∈ {RIDING_UPPER, FIRST_DIP, MID_BAND_BROKEN, CONSOLIDATING, UNKNOWN}
  
  # Volatility
  vol_state (SQUEEZE/NORMAL/EXPANDED), bb_width_pct, bb_width_pctl
  
  # Williams %R (50-period, threshold -20)
  wr_value, wr_in_momentum, wr_trend, wr_bars_since_cross50
  wr_phase ∈ {FRESH (≤3 bars), DEVELOPING (≤10), LATE (>10), NONE}
  
  # Combined gates
  entry_valid = wr_in_momentum AND daily_bias in (BULLISH, NEUTRAL)
  bb_squeezing = vol_state == SQUEEZE
  
  # Liquidity
  adv_cr = 20-day avg daily turnover in Rs Crore (Close × Volume / 1e7)
  
  # MFI (computed on daily resampled frame)
  mfi_value (0-100, None if unavailable)
  mfi_state ∈ {STRONG(>70), RISING(>55), NEUTRAL(>45), FALLING(>30), WEAK(≤30)}
  mfi_diverge = bearish divergence detected (3-step consecutive decline)
  mfi_reliable = adv_cr >= 5.0 AND mfi_series length >= 10

AnalyticsContext (modules/analytics.py):
  viability.score  (0–100, composite)
  viability.label  ∈ {STRONG, GOOD, CAUTION, AVOID}
  viability.sizing ∈ {FULL, HALF, QUARTER, SKIP}
  regime           ∈ {TRENDING, RANGING, ETF}
  room_pct         = distance from spot to BB upper (% room to ride)
  
  analyze_price_only(spot, price_signals, room_thresh=5.0) → AnalyticsContext
    Used by sector_rotation for each sector ETF's entry viability assessment.

Sector conviction scoring (DESIGNED, NOT YET IMPLEMENTED):
  conviction_score = adj_score
                   + entry_map[entry_label]    # STRONG:+3, GOOD:+1.5, CAUTION:-1, AVOID:-3
                   + mfi_map[mfi_state]        # STRONG:+1.5, RISING:+1, FALLING:-1, WEAK:-2
                   - (2.0 if mfi_diverge else 0)
  
  Sectors must be sorted by conviction_score, not adj_score.
  This is the single most important missing feature.
